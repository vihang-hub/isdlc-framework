## 1. Error Categories

### E-PROF-001: Profile File Parse Error

**Severity**: Warning
**Recovery**: Offer self-healing; fall back to `standard` if declined

| Code | Condition | Message | Action |
|------|-----------|---------|--------|
| E-PROF-001a | Malformed JSON | "Profile '{name}' has invalid JSON at line {n}: {detail}" | Offer to reformat |
| E-PROF-001b | Empty file | "Profile '{name}' is empty" | Offer to scaffold template |
| E-PROF-001c | Not an object | "Profile '{name}' root must be a JSON object, got {type}" | Offer to wrap in object |

### E-PROF-002: Schema Validation Error

**Severity**: Warning
**Recovery**: Offer field-level self-healing; fall back to `standard` if declined

| Code | Condition | Message | Action |
|------|-----------|---------|--------|
| E-PROF-002a | Missing required field | "Profile '{name}' missing required field: {field}" | Offer to add with default |
| E-PROF-002b | Wrong field type | "Profile '{name}': {field} should be {expected}, got {actual}" | Offer to cast |
| E-PROF-002c | Unknown field (typo) | "Profile '{name}': unknown field '{field}'. Did you mean '{suggestion}'?" | Offer to rename |
| E-PROF-002d | Unknown field (no match) | "Profile '{name}': unknown field '{field}' — will be ignored" | Log warning |
| E-PROF-002e | Unknown phase key | "Profile '{name}': unknown phase '{key}'. Did you mean '{suggestion}'?" | Offer to rename |
| E-PROF-002f | Invalid trigger value | "Profile '{name}': triggers must be an array of strings" | Offer to fix |

### E-PROF-003: Resolution Error

**Severity**: Info
**Recovery**: Ask user to choose from available profiles

| Code | Condition | Message | Action |
|------|-----------|---------|--------|
| E-PROF-003a | No trigger match | "No profile matches '{input}'. Available: {list}" | Ask user to choose |
| E-PROF-003b | Ambiguous match | "Multiple profiles match '{input}': {list}" | Ask user to disambiguate |
| E-PROF-003c | Named profile not found | "Profile '{name}' not found. Available: {list}" | Ask user to choose |

### E-PROF-004: Threshold Warning

**Severity**: Warning
**Recovery**: Inform only (or block if `enforce_minimum` is enabled)

| Code | Condition | Message | Action |
|------|-----------|---------|--------|
| E-PROF-004a | Coverage below recommended | "Profile '{name}' sets coverage to {n}% (recommended: 80%)" | Log warning |
| E-PROF-004b | Constitutional validation disabled | "Profile '{name}' disables constitutional validation" | Log warning |
| E-PROF-004c | Test iteration disabled | "Profile '{name}' disables test iteration" | Log warning |
| E-PROF-004d | Max iterations below recommended | "Profile '{name}' reduces max iterations to {n} (recommended: 5+)" | Log warning |
| E-PROF-004e | All gates disabled | "Profile '{name}' disables all gate checks — no quality validation will run" | Prominent warning |

### E-PROF-005: Filesystem Error

**Severity**: Warning
**Recovery**: Skip the affected tier; continue with available profiles

| Code | Condition | Message | Action |
|------|-----------|---------|--------|
| E-PROF-005a | Profile directory not found | (silent — expected for personal profiles) | Skip tier |
| E-PROF-005b | File read permission denied | "Cannot read profile '{path}': permission denied" | Skip file, log warning |
| E-PROF-005c | Self-healing write failed | "Could not save corrections to '{path}': {error}" | Fall back to `standard` |

### E-PROF-006: Enforcement Error

**Severity**: Block
**Recovery**: User must select a compliant profile

| Code | Condition | Message | Action |
|------|-----------|---------|--------|
| E-PROF-006a | Below enforced minimum | "Profile '{name}' violates enforced minimums: {details}. Select a compliant profile." | Block workflow start |

## 2. Error Propagation Strategy

```
Profile loading errors (E-PROF-001, E-PROF-002, E-PROF-005)
  --> Collected during loadAllProfiles()
  --> Invalid profiles excluded from registry
  --> Warnings surfaced at profile confirmation step
  --> Never propagate to gate checks (only valid profiles reach merge chain)

Resolution errors (E-PROF-003)
  --> Returned from matchProfileByTrigger() or resolveProfile()
  --> Handled by orchestrator (ask user to choose)
  --> Never propagate to gate checks

Threshold warnings (E-PROF-004)
  --> Generated by checkThresholdWarnings()
  --> Displayed at workflow start
  --> If enforce_minimum: converted to E-PROF-006 block

Enforcement errors (E-PROF-006)
  --> Block workflow start
  --> User must resolve before proceeding
```

## 3. Graceful Degradation

| Failure Scenario | Degradation Behavior |
|-----------------|---------------------|
| All profile directories missing | Use `standard` (hardcoded fallback) |
| Only personal profiles fail to load | Continue with project + built-in |
| Only project profiles fail to load | Continue with personal + built-in |
| Built-in profiles missing (framework corruption) | Use hardcoded `standard` defaults embedded in `profile-loader.cjs` |
| Selected profile fails validation | Offer self-healing, fall back to `standard` |
| `mergeRequirements()` throws on profile data | Catch, log, proceed with unmerged base requirements |
