# Test Data Plan: Ollama / Local LLM Support

**Feature:** REQ-0007-ollama-local-llm-support
**Phase:** 05-test-strategy
**Created:** 2026-02-14

---

## 1. Overview

All test data is generated programmatically within test files. No external test data files or fixtures are needed. The test data falls into 4 categories:

1. **Environment variable configurations** (process.env manipulation)
2. **Provider config objects** (JavaScript objects mimicking parsed YAML)
3. **File system fixtures** (temp directories with providers.yaml)
4. **Mock HTTP responses** (in-process HTTP servers)

## 2. Environment Variable Test Data

### Valid Configurations

| Scenario | ANTHROPIC_BASE_URL | ANTHROPIC_API_KEY | Expected Provider |
|----------|-------------------|-------------------|-------------------|
| Ollama via env | `http://localhost:11434` | (any or unset) | ollama |
| Anthropic via env | (unset) | `sk-ant-test123` | anthropic |
| Both set | `http://localhost:11434` | `sk-ant-test123` | ollama (base URL wins) |
| Neither set | (unset) | (unset) | Falls to tier 2+ |

### Boundary Values

| Scenario | ANTHROPIC_BASE_URL | Expected |
|----------|-------------------|----------|
| Uppercase localhost | `http://LOCALHOST:11434` | ollama (case-insensitive) |
| With path | `http://localhost:11434/v1` | ollama (contains localhost:11434) |
| Different port | `http://localhost:8080` | NOT ollama (no 11434) |
| Empty string | `""` | Falls to tier 2+ |
| Custom remote | `https://my-ollama.example.com:11434` | NOT ollama (not localhost) |

### Isolation Strategy

```javascript
// Save/restore in each test
let savedEnv;
before(() => {
    savedEnv = {
        ANTHROPIC_BASE_URL: process.env.ANTHROPIC_BASE_URL,
        ANTHROPIC_API_KEY: process.env.ANTHROPIC_API_KEY,
    };
});
after(() => {
    // Restore or delete
    for (const [key, val] of Object.entries(savedEnv)) {
        if (val === undefined) delete process.env[key];
        else process.env[key] = val;
    }
});
```

## 4. File System Fixtures

### Temp Directory Structure

```
/tmp/isdlc-test-XXXXXX/
  .isdlc/
    providers.yaml     (generated by writeProviderConfig)
  framework/
    claude/
      hooks/
        config/
          provider-defaults.yaml  (copied from source)
```

### Generation Strategy

```javascript
const os = require('os');
const fs = require('fs');
const path = require('path');

function createTestProject() {
    const tmpDir = fs.mkdtempSync(path.join(os.tmpdir(), 'isdlc-provider-test-'));
    const isdlcDir = path.join(tmpDir, '.isdlc');
    fs.mkdirSync(isdlcDir, { recursive: true });
    return { tmpDir, isdlcDir };
}

function createProviderFixture(tmpDir, provider, mode) {
    const yamlContent = `
defaults:
  provider: "${provider}"
  model: "sonnet"
active_mode: "${mode}"
providers:
  ollama:
    enabled: true
    base_url: "http://localhost:11434"
  anthropic:
    enabled: true
`;
    fs.writeFileSync(
        path.join(tmpDir, '.isdlc', 'providers.yaml'),
        yamlContent
    );
}

function cleanupTestProject(tmpDir) {
    fs.rmSync(tmpDir, { recursive: true, force: true });
}
```

## 5. Mock HTTP Server

### Health Check Mock

The health check mock is an in-process HTTP server that simulates Ollama's `/api/tags` endpoint.

```javascript
const http = require('http');

function createMockOllamaServer(behavior = 'healthy') {
    return new Promise((resolve) => {
        const server = http.createServer((req, res) => {
            if (req.url === '/api/tags') {
                switch (behavior) {
                    case 'healthy':
                        res.writeHead(200, { 'Content-Type': 'application/json' });
                        res.end(JSON.stringify({
                            models: [{ name: 'qwen3-coder' }]
                        }));
                        break;
                    case 'error':
                        res.writeHead(500);
                        res.end('Internal Server Error');
                        break;
                    case 'timeout':
                        // Never respond -- let timeout kick in
                        break;
                }
            } else {
                res.writeHead(404);
                res.end();
            }
        });

        server.listen(0, '127.0.0.1', () => {
            const port = server.address().port;
            resolve({ server, port, url: `http://127.0.0.1:${port}` });
        });
    });
}
```

### Mock Behaviors

| Behavior | HTTP Response | Test Purpose |
|----------|--------------|-------------|
| `healthy` | 200 + JSON body | Ollama is running and responsive |
| `error` | 500 | Ollama returns server error |
| `timeout` | No response | Ollama hangs or is overloaded |
| `connection_refused` | Server not started | Ollama is not running |

## 7. No External Dependencies

All test data generation uses only:
- `node:fs` for file operations
- `node:path` for path construction
- `node:os` for temp directory creation
- `node:http` for mock server
- Inline JavaScript objects for config data

No test data generators, faker libraries, or external fixtures are used. This complies with Article V (Simplicity First) and CON-001 (No New Dependencies).
